6. 생기부 작성

좌측 메뉴에서 생기부 작성(Write) 탭을 선택합니다. 상단 드롭다운에서 작성할 영역을 선택하면 학생 목록(행)과 활동(열)으로 구성된 격자 화면이 펼쳐집니다.

6-1. 격자 화면 구성 및 내용 입력

화면 구성

  • 행(Row): 해당 영역에 배정된 학생 목록 (학년·반·번호·이름 순 정렬)
  • 열(Column): 해당 영역에 속한 활동 목록
  • 셀: 학생 1명 × 활동 1개의 교차점 — 여기에 생기부 내용을 입력합니다.
  • 마지막 열 (합산): 해당 학생의 모든 활동 내용이 합산된 최종 문장과 총 바이트 수를 보여줍니다.
  1. 1 입력할 셀을 클릭하면 편집 모드가 활성화됩니다.
  2. 2 내용을 입력합니다. 글자를 입력하는 즉시 데이터베이스에 자동 저장됩니다. 별도의 저장 버튼이 없습니다.
  3. 3 Tab 키로 다음 셀로, Shift+Tab으로 이전 셀로 이동할 수 있습니다.

6-2. 바이트 수 실시간 표시 및 초과 경고

각 학생의 합산 열에는 현재까지 입력된 총 바이트 수가 실시간으로 표시됩니다. 영역에 설정한 최대 바이트를 초과하면 해당 행이 붉은색으로 강조되어 즉시 확인할 수 있습니다.

  • 정상: 바이트 제한 이내
  • 초과: 바이트 제한 초과 — 해당 학생 행이 붉은색으로 표시
💡

NEIS는 한글 2바이트, 영문·숫자 1바이트로 계산합니다. 이 프로그램도 동일한 방식으로 바이트를 계산합니다.

6-3. 셀 히스토리

셀에 입력한 내용은 변경될 때마다 히스토리로 누적 저장됩니다. 이전에 작성한 문장으로 돌아가고 싶을 때 활용하세요.

  1. 1 셀을 클릭한 뒤 히스토리 아이콘을 클릭합니다.
  2. 2 팝업에서 저장된 버전 목록을 확인합니다. 각 버전의 내용과 저장 시각이 표시됩니다.
  3. 3 원하는 버전의 복원 버튼을 클릭하면 해당 내용이 셀에 적용됩니다.

6-4. 스냅샷 (타임머신 저장)

스냅샷은 현재 DB 전체의 상태를 통째로 저장해두는 기능입니다. 대규모 수정 전에 스냅샷을 찍어두면 언제든 해당 시점으로 되돌릴 수 있습니다.

  1. 1 화면 상단의 스냅샷 버튼을 클릭합니다.
  2. 2 메모를 입력한 뒤 스냅샷 저장을 클릭합니다.
  3. 3 이전 스냅샷 목록에서 원하는 시점을 선택해 복원할 수 있습니다. 복원하면 현재 모든 데이터가 해당 시점으로 대체됩니다.
⚠️

스냅샷 복원은 복원 시점 이후의 모든 변경사항을 덮어씁니다. 중요한 작업 전에는 반드시 최신 스냅샷을 먼저 저장하세요.
